# 会议纪要生成

根据用户提供的会议文字稿（录音转写、速记稿等），生成一份结构清晰、信息完整的会议纪要。

## 处理流程

### 第一步：通读全文

通读用户提供的全部内容，理解会议的整体脉络——这场会议是关于什么的、谁在发言、讨论了哪些话题、达成了什么结论。

### 第二步：清洗内容

在生成纪要前，对原文进行必要的清洗：

- 去除口语停顿词（"嗯"、"啊"、"那个"、"就是说"、"然后…然后"等）
- 去除与会议主题无关的闲聊内容
- 合并同一发言人的连续表述
- 修正明显的转写错误（根据上下文推断正确内容）
- 保留所有实质性讨论内容

### 第三步：结构化输出

按照下方模板输出会议纪要。如果某项信息无法从文字稿中确定，**留空**即可，不要编造。

## 输出模板

严格按照以下格式输出：

```markdown
# 会议纪要

## 一、会议基本信息

| 项目 | 内容 |
|------|------|
| 会议主题 | （填写） |
| 会议时间 | （填写，格式：YYYY年MM月DD日 HH:MM） |
| 会议地点 | （填写） |
| 主持人 | （填写） |
| 记录人 | （填写） |
| 参会人员 | （列举所有参会者姓名及角色） |
| 缺席人员 | （填写） |

## 二、会议目标

### 会议目的
（简述本次会议要解决什么问题或达成什么目标）

### 背景信息
（提供理解本次会议所需的前置背景，如项目阶段、上次会议结论等）

### 预算/资源
（如涉及预算或资源分配，在此说明；不涉及则留空）

## 三、会议内容

### 议题一：[议题名称]
**讨论内容：**
（围绕该议题，各方发表了什么观点，讨论了什么方案）

**结论/共识：**
（该议题达成的结论或共识是什么；未达成则注明"未达成一致"）

### 议题二：[议题名称]
**讨论内容：**

**结论/共识：**

（根据实际议题数量继续添加）

## 四、行动项

| 序号 | 行动事项 | 负责人 | 截止时间 | 备注 |
|------|----------|--------|----------|------|
| 1 | （具体要做什么） | （谁负责） | （何时完成） | （补充说明） |
| 2 | | | | |

## 五、待决事项

（列出本次会议未解决、需后续讨论的问题）

## 六、下次会议

| 项目 | 内容 |
|------|------|
| 是否已安排 | （是/否） |
| 预计时间 | （填写） |
| 预计议题 | （列举） |
```

## 质量要求

- **完整性**：不遗漏任何实质性讨论内容和决策
- **准确性**：忠实于原文，不添加文字稿中没有的信息
- **简洁性**：去除冗余，每个议题用精炼的语言概括核心内容
- **可追溯性**：重要观点和决策应能对应到原始发言
- **诚实性**：信息不确定时宁可留空，绝不编造

## 示例

**输入（原始文字稿）：**

```
嗯，好，那个我们今天开个短会啊，主要是聊一下这个Q3的市场推广方案。
小李你先说一下你那边的情况。

嗯好的，就是我们这边市场部的话，目前Q3的预算大概在50万左右，
然后我们计划是主要在线上做投放，就是抖音和小红书这两个渠道。
然后那个时间的话，我们从7月中旬开始，大概持续到9月底。

好的，那我这边产品说一下啊，我们7月20号会上一个新版本，
里面有个比较大的功能更新，就是那个用户推荐系统。
所以我们建议推广的时间最好配合这个版本发布来走。

对，这个没问题，那我们定一下具体的timeline。
小李你回头整理一个详细的投放计划，下周三之前发出来。
然后产品那边，你们7月18号之前给一个功能亮点清单，
这样小李可以提前准备素材。

嗯，还有那个预算的话，50万是不是有点紧？
要不这样，小王你去看一下能不能再追加10万的预算，
下周给我一个答复。

好的好的，那就这样，大家还有别的事吗？没有的话今天就到这。
```

**输出：**

```markdown

| 项目 | 内容 |
|------|------|
| 会议主题 | Q3市场推广方案讨论 |
| 会议时间 | |
| 会议地点 | |
| 主持人 | |
| 记录人 | |
| 参会人员 | 小李（市场部）、产品负责人、小王 |
| 缺席人员 | |

## 二、会议目标

### 会议目的
讨论并确定Q3市场推广方案的方向、预算和时间安排。

### 背景信息

### 预算/资源
市场部Q3预算约50万元，计划追加10万元（待确认）。

## 三、会议内容

### 议题一：Q3市场推广渠道与计划
**讨论内容：**
- 市场部（小李）提出Q3预算约50万元，计划通过抖音和小红书进行线上投放，时间从7月中旬持续到9月底。
- 产品方提出7月20日将发布新版本，包含用户推荐系统等重要功能更新，建议推广时间配合版本发布。

**结论/共识：**
- 推广渠道确认为抖音+小红书线上投放
- 推广时间配合7月20日产品版本发布

### 议题二：预算评估
**讨论内容：**
- 有参会者提出50万预算可能偏紧，建议追加预算。

**结论/共识：**
- 由小王评估是否可追加10万预算。

## 四、行动项

| 序号 | 行动事项 | 负责人 | 截止时间 | 备注 |
|------|----------|--------|----------|------|
| 1 | 整理详细投放计划并发出 | 小李 | 下周三前 | |

## 五、待决事项

- 追加预算是否获批（待小王确认）

## 六、下次会议

| 项目 | 内容 |
|------|------|
| 是否已安排 | 否 |
| 预计时间 | |
| 预计议题 | |
```

## 注意事项

- 如果文字稿内容非常长，可以先将内容按主题分段，再逐段处理
- 对于多人同时发言、录音不清晰导致的无法辨认内容，用 `[听不清]` 标注
- 如果文字稿包含多语言混杂，保留原文用语（如英文术语不强制翻译）
- 会议中出现的分歧意见要如实记录，不要为了"达成一致"而粉饰
- 行动项必须包含三个要素：**谁** + **做什么** + **何时完成**，缺一不可；缺失时留空
