## Token 词元
用户输入Prompt（自然语言）分割为多个token，每个token转成一个token id，这些token id 作为模型的输入
编码器，将文字编码为token id，decode解码器，将token id 解码为文字

大模型处理的最小单元不一定是完整的词，可能是子词，字符，甚至是标点符号，用词元能更精准的表达它是模型计算和计费的通用最小处理单位，而非语言学意义上的词，有利于模型计算
例：unhappiness 可以被分解为，un，happi，ness
llm通常不会把它当成一个完整的词，而是将其分解为多个词元
tokenization 分解为 token ization
如果只认完整的词，模型需要记住几十万个英文单词和几百万个中文词汇，计算量会很大，把词切成子词，模型只需要掌握几万个基础积木（词元）

token查找表，size就不会那么大，运算效率就高
openai 事实标准 国内有千问

中文：我爱人工智能，自然语言处理很有趣
word：‘我’，’爱’，’人工智能’，’，’，’自然语言处理’，’很’，’有趣’

把词元理解为llm的货币
llm只会做一件事情，预测下一个词

这个词元的语义向量，某一页大写特写，存在神经网络结构中

自回归生成：预测一个词，再根据这个词，继续预测下一个词

## Embedding
语义向量 pre-trained 神经元
第一步：用tokenid变为一个embedding向量，这个向量表示了词元的语义信息
你 -> tokenid(57688)没有任何含义
你 -> 好 在高维向量空间中距离比较近
语义 + 距离计算
向量是多维空间里的一个坐标点，向量是有方向的，可以计算距离
语义相近的词，他们的向量距离比较近

第二步：用embedding向量表示的词元，计算出词元之间的距离，这个距离表示了词元之间的相似度
llm 把编号查表转成一个高维的向量，这个过程就叫embedding编码
llm内部有个巨大的向量查找表
模型拿到tokenid 你（57668） 直接去57668的区域把对应的向量取出来，作为你的embedding向量

国王 王后 这两个点的距离很接近
国王 - 男性的向量 + 女性的向量 = 王后

模型的训练就是会构建这样的几何结构的空间坐标系

## 位置编码
”我咬了你“
“你咬了我“
顺序在当前上下文非常重要
光有语义的理解还不够，相同的四个字，顺序不同，含义也不同
embedding 不携带位置信息，我们就给每个向量叠加一个位置编码，告诉llm这个词属于句子的第几个位置
每个token 携带两类信息:一个是语义信息，一个是位置信息

## 模型怎么理解上下文?  it?
不止光靠一个词的向量
代词(it)怎么向量计算？ 根据上下文将animal的语义带入it
The animal didn't cross the street, because it was too tired.
it是指代animal还是street
llm引用的机制是 self-attention（自注意力）
每一个embedding分成三个部分：查询向量、键向量、值向量
Q Query 查询：我这个词「想找谁、要什么信息」
K Key 键：每个词对外展示的「身份标签」
V Value 值：每个词携带的「真实语义内容」
Q_it × K_animal → 分数很高（语义高度匹配：生物、有疲惫属性）
Q_it × K_street → 分数极低（街道无生命，不可能累）
