## Agent 智能体
和 workflow的逻辑完全不一样，它不走在一条预设的轨道上，而是站在一个开放的空间里，自己去探索怎么去做（智能）
从技术角度来看，一个Agent往往具备三个核心能力
第一个能力：感知环境
通过输入文本，图像，数据来理解任务，有哪些工具可以调用，当前情况如何。如何分步骤，如何规划
第二个能力：规划路径
不是预先定义好的，而是动态去生成任务链
第三个能力：执行任务
去调用工具，完成子任务，并且不断地去调整

## 举例
你让Agent去规划旅行，它可能会先去搜一下机票和酒店的价格，再去查询天气和签证需求，然后结合你的偏好和预算，动态权衡方案
整个过程之中，它可能走的路径和你预期的完全不同，这就是Agent和Workflow的关键区别，它有自主性和适应性，它不会被一个固定的流程限制，而是像一个思考中的个体
本质的区别在于：workflow是确定性的执行，而Agent是一个不确定的探索。workflow就像一条高速公路，路线清晰固定，效率很高
Agent 更像一个司机，它知道目的地，但可以随时改变路线（感知环境），尝试新的路线，甚至在目的地发生变化时，也可以重新规划路线

所以，从产业角度，workflow，更适合流程化，重复性的任务，比如审批流程，数据清洗，客服AI 
Agent 更适合动态的场景，比如复杂搜索，策略规划

从技术栈的领域，workflow更依赖于规则，图形化编排，流程引擎
Agent 更依赖于llm推理能力，记忆能力，工具调用等

未来AI工程中，将会集合Agent和Workflow的双重能力
底层是workflow去提供稳定性和可控性，上层是Agent提供灵活性和智能性

Workflow是骨架，Agent是Workflow的大脑
workflow缺点是没什么创造力，Agent有，但是不太可控，不可预测
一起结合，即高效又智能
