# 轻松上手 Vibe coding

想去用AI生成代码，但是AI生成的要不是“幻觉代码”，就是“错误代码”。看起来像那么回事，一跑就出错，要么就是屎山代码，能跑，但是想修改都不知道从哪里下手 

这个指南，不是怎么写代码，而是我们和AI协作写出靠谱的代码
- 新手 自然语言编程，不要急于写代码，先思考，规划md文件，每次prompt都会自动规划，保持一致性，有整体的约束

## 操作的核心方法
- 第一步 ：规划就是一切
不要让AI一开始就写代码，先强制它去读懂你的技术栈和项目规划(/init)
就好像你不会让新来的员工直接干活，得先和熟悉公司业务流程一样，这样AI生成代码的逻辑清晰多了
新手把codex，cc 工具，老手把他们当伙伴，同事，助手

```
遵守胶水编程思维：优先使用成熟方案，避免凭空造逻辑

```
背后逻辑
先划定边界，防止AI 擅自多加功能，这就是造成无限膨胀成屎山代码的原因
强制模块拆分(好读好维护，AI生成的代码我们都要审核)
引导ai复用独立组件，预先规定数据结构吗，从根源上减少AI参数，字段幻觉,字段不能让AI做主
先生成规划，审核规划，更新规划
这份规划将伴随我们的项目开发完整周期

第二个阶段：”胶水“编程规则
能抄不写（github 经典，优质，经过验证的代码），能连不造（胶水）
指挥AI 去找那些已经被验证过千万次的成熟组件，拼乐高一样，把它组装起来
胶水是一个比喻，可以粘合，少量AI生成，产生幻觉的可能性大大降低

胶水本身不创作零件，只负责把现成零件连接在一起，你只负责写衔接，调用，流转的粘合代码，把各个模块联通

轮子别人造好，你只做胶水，胶水不生产零件，只联通零件
