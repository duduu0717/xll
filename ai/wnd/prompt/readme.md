## hashTable 哈希表
- 哈希表是一种基于键值对存储数据的数据结构
-key是唯一的，进行哈希（计算函数），得到一个索引
-索引指向一个存储位置，存储着对应的value
-当需要根据key查找value时，先对key进行哈希计算，得到索引，直接从存储位置获取value

## dict的特点
-查找和插入的速度很快，不会随着key的增加而变慢
-需要占用大量内存，内存消费多

## list特点相反
-查找和插入的速度很慢，会随着key的增加而变慢
-不需要占用大量内存，内存消费少

dict可以用在需要高速查找的地方

## unhashable type
-dict 的key必须是不可变的，可哈希的，否则会报错
数组可变，内容可以随时添加
hash key 必须要唯一
dict 靠key计算value的存储位置，如果key可变，每次计算出来的结果可能不同，dict会混乱
这个通过key计算位置的算法为哈希算法

## Set
和dict类似，也是一组key的集合，但不存value
由于key不能重复，所以在set中没有重复的key

set和dict唯一的区别在于有没有存储对应的value，set原理和dict相同

## 再谈不可变对象

str是不可变对象，而list是可变对象

str是变量，可以被重新赋值
’abc‘才是字符串对象 replace作用在字符串对象’abc‘上，并没有改变‘abc’内容，返回了一个新的‘Abc’
所以对于不可变对象，调用任意方法，也不会改变该对象的内容

变量提升时，函数优先于其他变量
如果定义了两个相同函数，最终后定义的函数会覆盖先定义的函数，生效的只有后定义的函数
