# Agent 智能体 ？
字面意义上，AI会自己干活
这么理解没有问题，但是遮住了一个更重要的问题

Agent和普通的ai对话，到底有什么区别
结构导致Agent的工作方式
- 对话结构来分析
普通对话，你问它答，一问一答就结束
案例：帮我写一封邮件
llm写完，任务就终止
问答机器，输出一次没有后续
Agent不一样，它有一个持续运转的结构
你给它一个任务，它会去拆解任务，决定下一步，去调用工具，查看结果
这个过程是一个循环，直到任务完成，或者它自己判断（超出循环次数，token上限，相同结果次数，失败了）

## Agent的工作方式，核心三个动作
- 第一个动作思考 Reason
- 第二个动作执行 Act
- 第三个动作观察 Observe

这三步是一轮循环，观察后再回到思考，再执行，再观察，循环往复，这个循环就是ReAct循环
ReAct （Reasoning + Act + Observe）是一套标准的Agent执行流程/Agent工作框架，不是langchain那种大型的开发库，是Agent通用的循环工作标准

## demo
帮我分析竞品，然后去写一份报告，Agent会怎么做

- 第一轮循环
思考 Reason：我现在啥竞品数据都没有，直接写报告只能瞎编。第一步得先搞清楚这个行业有哪几个主要竞品。
执行 Act：调用联网搜索，查这个行业主流竞品名单。
观察 Observe：搜到 3 个竞品名字，只有简单介绍，没有价格、功能、用户评价，资料不够写报告。
- 第二轮循环
思考 Reason：光知道名字没用，我要查这三家的定价、有什么特色功能、用户吐槽的缺点。
执行 Act：再次搜索三家竞品的价格套餐、功能对比、网上用户好评差评。  
观察 Observe：拿到完整对比数据：各自收费、独有功能、短板、用户痛点，资料全部齐了。
- 第三轮，任务收尾
思考 Reason：所有需要的数据都搜集完了，不用再上网查了，可以整合信息写完整报告。
执行 Act：整理全部资料，分段写出竞品概况、优劣对比、市场机会、总结建议，生成完整报告。
观察 Observe：报告写完，任务结束，交给用户

Agent最核心的动作，就是Tool use
工具就是Agent的手和脚，没有工具，它只能在脑子里转，转完之后，还是只有文字
常见的工具：
- 搜索工具，能上网查实时信息
- 代码执行器，能运行代码，能看结果
 Anthopic 最牛逼的Agent企业，代码是最标准的，又完善工程化验收机制的，ai测试agent
- 文件读写I/O
- 浏览器的操控browser 打开网页，点击提交
 manus
- api调用
工具越多，Agent能干的事情越多
很多Agent产品，调用了很多工具，工具的覆盖范围，直接决定Agent的能力边界，选择Agent的核心
