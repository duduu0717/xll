### 总结
-在执行的过程中，若使用了未声明的变量，会报错 xxx is not defined
 很明显的代码错误
-在一个变量定义之前使用它，不一定会报错（let，const会报错 死区），但该变量的值是undefined，而不是定义时的值
-在一个函数定义之前使用它，不会报错，且函数执行正确

后两个错误难理解，表面js代码不是一行一行执行的
不只有执行阶段，还有编译阶段，编译阶段会为执行阶段做准备，准备好执行上下文

## 变量提升发生在编译阶段（hoisting）
-先了解编译阶段执行阶段和作用域，执行上下文，调用栈
-js 是脚本语言，弱类型动态语言，它没有独立的编译阶段
 就在运行前的一瞬间

 ## 所谓的变量提升，是指在js代码执行过程中，js引擎（Chrome v8引擎），把变量声明部分和函数的声明部分提升到代码开头，变量提升后，会给变量设置默认值，就是undefined

 模拟变量提升
 函数和变量在执行之前都是提升到了代码开头

 ## js代码的执行流程
 “变量提升”意味着变量和函数的声明会在**物理**层面移动到代码的最前面
 并不准确，实际上变量和函数声明在代码的位置是不会改变的，而且是在编译阶段被js引擎放入内存中

 js的代码在执行前，需要被js引擎编译，编译完成后才会进入执行阶段

 ## 编译阶段 

词法环境的变量没有变量提升（提升到死区），在声明之前的词法环境中的变量处于死区中，不能使用，如果在声明前使用，会报错

输入一段代码，经过编译阶段，会生成两段内容：
执行上下文和可执行代码

调用一个函数，就会进入这个函数的上下文，找到上下文的变量环境，词法环境
