## Fragment 组件
- 它可以作为容器，内部挂载子元素们（dom树功能）
- 一次性的挂载到页面 #root，fragment 元素就会功成身退

## 异步
set是异步调度更新，不会立即修改count 调用后当前作用域count仍然是初始值，日志打印旧数据 等本轮代码执行完毕，组件重新渲染才拿到新值

组件里的状态比较多，xyz坐标，移动，多个状态同时改
react 状态的更新是靠react 组件函数的重新运行来实现的
三次调用setCount(count + 1)本质都是0 + 1 = 1 三次都是一样的，只执行最后一次，目的是性能优化
如果非要达到+3的效果，多次执行，每一次都拿到最新的状态值
传函数，基于当前状态，返回全新状态，不再是闭包引用
