## 链表
- 数组增加可能会扩容 arr.length>=capacity 扩容和初始化才有内存申请分配的开销,其他时刻都没有
- 链表 每次都申请,开销比较平均
数据的规模
小就用数组 大就用链表
数组的优势是遍历,按索引取元素,缺点是增删,需要移动位置和申请空间
链表优势是增删(前驱节点,不要坐过站)缺点是找到相应的节点必须从头开始遍历

链表和数组相似的,他们都是**有序**的列表(list),都是线性结构(有且仅有一个前驱节点和后继节点)
 数组 array索引
 链表 linked list : head -> ...node -> tail

链表中,数据单位节点,指针
节点和节点的分布是不连续的,是离散的
js用对象字面量表示链表节点
{
  val:1
  next:{
    val:2
    next:null
    }
  }
}

为什么会有head节点?
访问链表中的任何一个元素,从起始节点开始(val 空),逐个访问,一直访问到目标节点,tail节点next为null

## 链表元素的添加
本质上是对next指针操作
找到前驱节点
新节点的 next 指向前驱原来的下一个节点
前驱节点的 next 改成指向新节点

## 删除
假设数组的长度是n,因为增加/删除操作导致需要移动的元素的数量,随着数组的长度n的增大而增大,呈一个线性关系
增加删除操作对应的时间复杂度是O(n)

- js数组特性

js数组未必是真正的数组
如果每项类型一致,纯数组,连续内存
如果类型不同,一段非连续内存,此时js数组不在具有数组的特征,底层是使用哈希分配内存空间
数字下标 = 对象的键 (key)，元素 = 值 (value)，这就是哈希映射。

链表有高效的增删操作
O(n) 固定的
明确要插入或者删除的目标位置,接着只需要改变目标节点和前驱后继节点指针的指向 O(1)
第一步：找位置（遍历）→ O (n)
第二步：改指针（真正增删）→ O(1)
麻烦的访问操作O(n)
